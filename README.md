Telecom Customer Churn Analysis
This project analyzes customer churn in the telecom industry using SQL for data processing and Power BI for visualization. The goal is to identify why customers leave and highlight the potential revenue at risk.

Tools Used
SQL (MySQL Workbench): Data cleaning, exploratory data analysis (EDA), and customer risk segmentation.
Power BI: Interactive dashboard design and data visualization.

Repository Structure
/SQL_Scripts: Contains the SQL queries used to clean the data, calculate churn rates, and segment users.
/Dashboard: Contains the Power BI file and screenshots of the final dashboard.

Key Insights
Overall Churn: 26.5% of customers churned, putting $139.13K in revenue at risk.
Contract Type: Month-to-month contracts have the highest churn rate by a wide margin.
Tenure: Newer customers (0-1 year) are much more likely to leave than long-term customers.
Payment Method: Customers paying by electronic check churn significantly more than those using automated payments (like credit cards or bank transfers).
